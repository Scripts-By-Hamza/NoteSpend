1. EXECUTIVE SUMMARY
================================================================================

NoteSpend is an offline-first web application that combines note-taking and
expense tracking in a single, unified platform. The application addresses the
common pain point of users needing to switch between multiple apps to manage
their daily notes and financial transactions.

Key Differentiator: Smart Linking System that allows users to connect notes
with related expenses, enabling use cases like trip budgeting, project expense
tracking, and event planning.

Target Launch: Q2 2026 (MVP)
Platform: Progressive Web App (PWA)
Primary Markets: Students, Freelancers, Small Business Owners, Travelers

--------------------------------------------------------------------------------
6.1 NOTES MODULE
--------------------------------------------------------------------------------

Feature Set:

1. Create Note
   - Required Fields: Title
   - Optional Fields: Description (rich text)
   - Auto-generated: Timestamp, unique ID
   - Character Limits: Title (100 chars), Description (10,000 chars)

2. Edit Note
   - Modify title, description, tags
   - Update linked expenses
   - View edit history (future enhancement)

3. Delete Note
   - Confirmation dialog
   - Option to keep/remove linked expenses
   - Soft delete with 30-day recovery period

4. Organize Notes
   - Tags: Create custom tags (max 20 per note)
   - Pin: Mark important notes (unlimited)
   - Sort: By date, title, or last modified
   - Filter: By tag, date range, linked expenses

5. Search Notes
   - Full-text search across title and description
   - Filter by tags
   - Search within date ranges
   - Instant results (< 100ms)

6. Attachments (Phase 2)
   - Supported formats: Images (JPG, PNG), PDFs, documents
   - Size limit: 10MB per file, 50MB total per note
   - Storage: IndexedDB as Base64

7. Linked Expenses
   - View all expenses linked to note
   - Quick add expense from note detail page
   - Total expense summary for note
   - Filter note by expense amount range

Acceptance Criteria:
- Note creation takes < 2 seconds
- Search returns results in < 100ms for 1000+ notes
- Offline mode supports all CRUD operations
- No data loss during offline/online transitions


--------------------------------------------------------------------------------
6.2 EXPENSE MODULE
--------------------------------------------------------------------------------

Feature Set:

1. Add Expense/Income
   - Type: Toggle between Expense/Income
   - Amount: Numeric input (up to 2 decimal places)
   - Category: Dropdown selection
   - Date: Date picker (defaults to today)
   - Description: Optional text (500 chars)
   - Receipt: Optional image upload
   - Linked Note: Optional note selection

2. Expense Categories (Customizable)
   Default Categories:
   - Food & Dining
   - Transportation
   - Shopping
   - Bills & Utilities
   - Entertainment
   - Healthcare
   - Education
   - Travel
   - Rent/Mortgage
   - Other

3. Income Categories (Customizable)
   Default Categories:
   - Salary
   - Freelance
   - Business
   - Investment
   - Gift/Bonus
   - Other

4. Edit/Delete Expense
   - Full edit capability
   - Confirmation before deletion
   - Update linked note relationships

5. Filter & Search
   - Date range selection
   - Category filter (multi-select)
   - Amount range filter
   - Linked note filter
   - Type filter (expense/income/both)

6. Receipt Management
   - Camera capture or file upload
   - Image preview
   - Crop/rotate capability (Phase 2)
   - OCR text extraction (Phase 3)

7. Monthly Summary Dashboard
   - Total Income (current month)

   - Net Balance (income - expenses)
   - Category-wise breakdown
   - Daily average spending
   - Comparison with previous month

Acceptance Criteria:
- Expense entry takes < 3 seconds
- Filters apply in real-time (< 200ms)
- Receipt images load in < 1 second
- Monthly calculations accurate to 2 decimal places

6.3 SMART LINKING SYSTEM
--------------------------------------------------------------------------------

Core Concept:
Enable bidirectional relationships between Notes and Expenses to provide
contextual financial tracking.

Relationship Rules:
- One Note → Many Expenses (1:N)
- One Expense → One Note (N:1)
- Optional relationship (not mandatory)

Use Cases:

Use Case 1: Trip Planning
Note: "Lahore Weekend Trip - March 2026"
Linked Expenses:
  - Fuel: PKR 3,000
  - Hotel: PKR 12,000
  - Food: PKR 4,500
  - Shopping: PKR 2,000
Total Trip Cost: PKR 21,500

Use Case 2: Client Project
Note: "Website Design - Client ABC"
Linked Expenses:
  - Stock Photos: USD 50
  - Hosting: USD 15
  - Domain: USD 12
Total Project Cost: USD 77

Use Case 3: Event Planning
Note: "Sister's Birthday Party"
Linked Expenses:
  - Venue: PKR 8,000
  - Catering: PKR 15,000
  - Decorations: PKR 3,500
  - Gifts: PKR 4,000
Total Event Cost: PKR 30,500

Features:
1. Quick Link Creation
   - Link expense to note during creation
   - Add expenses from note detail page
   - Bulk link multiple expenses to note

2. Link Management
   - View all linked items
   - Unlink individual items
   - Transfer links to different note

3. Aggregated Views
   - Total linked expenses per note
   - Notes with highest expenses
   - Unlinked expenses view

4. Smart Suggestions (Phase 3)
   - Auto-suggest notes based on expense category
   - Detect recurring patterns
   - Budget alerts for linked items

Acceptance Criteria:
- Linking/unlinking takes < 1 second
- Link updates sync across both modules instantly
- No orphaned links after deletions

8.1 TECHNOLOGY STACK
--------------------------------------------------------------------------------

Frontend Framework:
- React 18.x (with Vite)
  Rationale: Component reusability, large ecosystem, fast development

Build Tool:
- Vite 5.x
  Rationale: Fast HMR, optimized builds, modern tooling

Styling:
- Tailwind CSS 3.x
  Rationale: Utility-first, responsive, small bundle size
- CSS Modules (for complex components)

Icons:
- Lucide React
  Rationale: Consistent, customizable, tree-shakeable

State Management:
- React Context API + useReducer
  Rationale: Sufficient for app complexity, no external dependencies
- Local state for UI interactions

Routing:
- React Router v6
  Rationale: Standard, well-documented, supports nested routes

Database:
- IndexedDB
  Rationale: Large storage capacity, structured data, async API
- Wrapper Library: Dexie.js
  Rationale: Simplified API, better error handling, hooks support

Charts:
- Recharts
  Rationale: React-native, declarative, responsive

Date Handling:
- Day.js
  Rationale: Lightweight (2KB), Moment.js alternative, i18n support

Utilities:
- uuid: Unique ID generation
- file-saver: File export functionality
- papaparse: CSV parsing/generation

PWA Features:
- Workbox (Service Worker library)
- Web App Manifest
- IndexedDB for offline data

Form Management:
- React Hook Form
  Rationale: Performance, minimal re-renders, easy validation

Validation:
- Zod
  Rationale: TypeScript-first, runtime validation, great DX

Testing (Phase 2):
- Vitest (unit tests)
- React Testing Library (component tests)
- Playwright (E2E tests)

8.2 DATA MODELS
--------------------------------------------------------------------------------

Note Schema:
{
  id: string (UUID),
  title: string (required, max 100 chars),
  description: string (optional, max 10000 chars),
  tags: string[] (max 20 tags),
  pinned: boolean (default: false),
  createdAt: timestamp (ISO 8601),
  updatedAt: timestamp (ISO 8601),
  attachments: [
    {
      id: string (UUID),
      name: string,
      type: string (MIME type),
      size: number (bytes),
      data: string (Base64) | Blob
    }
  ],
  linkedExpenseIds: string[] (UUIDs),
  isDeleted: boolean (default: false),
  deletedAt: timestamp | null
}

Expense Schema:
{
  id: string (UUID),
  type: 'expense' | 'income',
  amount: number (decimal, 2 places),
  currency: string (ISO 4217 code, e.g., 'USD'),
  category: string,
  date: timestamp (ISO 8601),
  description: string (optional, max 500 chars),
  receiptImage: {
    id: string (UUID),
    data: string (Base64) | Blob,
    type: string (MIME type)
  } | null,
  linkedNoteId: string (UUID) | null,
  createdAt: timestamp (ISO 8601),
  updatedAt: timestamp (ISO 8601),
  isDeleted: boolean (default: false),
  deletedAt: timestamp | null
}

Category Schema:
{
  id: string (UUID),
  name: string,
  type: 'expense' | 'income',
  icon: string (icon name),
  color: string (hex code),
  isDefault: boolean,
  createdAt: timestamp (ISO 8601)
}

Settings Schema:
{
  currency: string (default: 'USD'),
  dateFormat: string (default: 'DD/MM/YYYY'),
  theme: 'light' | 'dark' | 'auto' (default: 'auto'),
  fontSize: 'small' | 'medium' | 'large' (default: 'medium'),
  firstDayOfWeek: number (0-6, default: 0 for Sunday),
  showDeletedItems: boolean (default: false)
}

8.3 STORAGE & PERFORMANCE
--------------------------------------------------------------------------------

IndexedDB Structure:

Database Name: NotesSpendDB
Version: 1

Object Stores:
1. notes
   - Key Path: id
   - Indexes: createdAt, updatedAt, pinned, tags, isDeleted

2. expenses
   - Key Path: id
   - Indexes: date, category, type, linkedNoteId, isDeleted

3. categories
   - Key Path: id
   - Indexes: type, name

4. settings
   - Key Path: key
   - No indexes

Storage Quotas:
- Target: 50MB - 100MB
- Maximum: 500MB (device dependent)
- Alert user at 80% capacity

Performance Targets:
- Initial load: < 2 seconds
- Note creation: < 500ms
- Expense creation: < 500ms
- Search results: < 100ms (up to 1000 items)
- Chart rendering: < 1 second
- Offline sync queue processing: < 3 seconds

Optimization Strategies:
1. Lazy loading: Load lists in batches (20 items initially)
2. Virtual scrolling: For large lists (>100 items)
3. Memoization: Cache expensive calculations
4. Debounced search: 300ms delay
5. Image compression: Resize uploads to max 1920px
6. Code splitting: Route-based chunks
7. Tree shaking: Remove unused code
8. Service Worker caching: Cache static assets

- Implement auto-save (every 5 seconds)

- Progressive enhancement strategy

Impact: MEDIUM | Probability: MEDIUM

- Attachment size limits
